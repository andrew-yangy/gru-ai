# Brainstorm Synthesis — gruAI Marketing Launch

## Convergence (all 3 agents agree)

1. **Repo name**: Rename GitHub from `agent-conductor` to `gruai` (no hyphen). Keep npm as `gru-ai` (already published).
2. **Brand conflict**: Gru.ai is a funded ($5.5M) AI coding agent company (BabelCloud). They pivoted to gbox.ai but still own the name. Monitor for trademark issues, prepare backup name.
3. **Hero visual**: The pixel-art office GIF is THE differentiator. Lead everything with it. Verify docs/assets/demo.gif is current — re-record if stale.
4. **README first**: Polish README before any external launch. Lead with GIF, not architecture docs.
5. **No paid ads**: Zero organic foundation means paid spend has nothing to amplify.
6. **Landing page = simple**: Static HTML, no framework. GitHub Pages from docs/.

## Key Disagreements

### 1. Channel Priority: 小红书 first vs Reddit/HN first

**Priya (CMO)**: 小红书 first — visual-first platform, AI dev content is trending there, zero competition for Claude Code visualization tools in Chinese dev community. 3x/week cadence with content pillars.

**Marcus (CPO) + Sarah (CTO)**: Reddit/HN first — gruAI users are English-speaking Claude Code developers. r/ClaudeCode (4200+ weekly contributors) is the exact audience. 小红书 audience skews lifestyle/consumer tech, not CLI dev tools. No evidence Chinese devs are our highest-conversion segment.

**Rebuttal impact**: Marcus's critique landed — "what comparable developer CLI tool got traction on 小红书?" is unanswered. Sarah added: "earn that complexity with evidence first."

### 2. Scope: Full marketing strategy vs minimal viable launch

**Priya**: 5 sequential phases, 3x/week content calendar, content pillars, measurement framework.

**Sarah + Marcus**: Two projects max. Ship rename + README + landing page (1 afternoon), then ONE Reddit post + ONE Show HN post. Measure. Decide what's next based on signal.

**Rebuttal impact**: Sarah's critique landed — "this is a marketing plan for a funded startup, not an open-source framework with zero users." Priya's counter: "if it's not in a project with tasks/DOD, nobody will own it and it'll sit in a backlog forever."

### 3. AI-generated video vs real screencasts

**Sarah**: AI video tools not mature enough for quality bar. Record real demo screencasts.

**Priya**: Didn't explicitly defend AI video but proposed video content as a pillar.

**CEO's directive**: Specifically asked about AI-generated video for 小红书 ("想办法用AI生成视频和文章").

### 4. Marketing content as engineering project

**Sarah**: Marketing content is NOT engineering. Hand off as a document for CEO/Priya to execute. Content calendars belong in Notion, not .context/.

**Priya**: Needs same rigor as engineering — tasks, DOD, reviewers — or it'll never ship. "Who records screencasts? When? With what script? There's no plan, just a veto."

## Proposals That Survived Challenge

- Rename + README polish (unchallenged)
- Static HTML landing page (unchallenged, but timing disputed)
- Lead with pixel-art office visual (unchallenged)
- Position on visibility/visualization, not price (unchallenged)
- Target r/ClaudeCode as first English channel (unchallenged)
- Do NOT post on r/artificial or r/LocalLLaMA (unchallenged)
