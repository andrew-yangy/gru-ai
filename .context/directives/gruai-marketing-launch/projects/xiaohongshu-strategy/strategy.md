# 小红书 Content Strategy -- gruAI

## 1. Content Pillars

### Pillar A: AI Agents Running a Company (AI智能体自动化公司)

**Core narrative:** gruAI is not just another AI coding tool. It is a
framework where named AI agents -- CTO, CPO, CMO, COO -- each own a domain,
challenge decisions, propose initiatives, and execute work autonomously. The
CEO reviews outcomes, not code.

**Emotional hook:** "What if your entire dev team was AI -- and they argued
with each other?"

**Example topics:**
- How Morgan (COO) plans a feature from scratch -- directive to deployment
- Sarah (CTO) rejecting a plan because it violates architecture principles
- The challenge mode: agents push back on CEO decisions with reasoning
- A day in the life of an autonomous AI company: Monday scout, Tue--Thu build,
  Friday report
- CEO spends 45 minutes/week -- what the rest of the time looks like

**Format affinity:** Screencast (showing pipeline execution in terminal),
carousel (step-by-step pipeline flow diagrams)

### Pillar B: Pixel-Art Office Visualization (像素风办公室)

**Core narrative:** gruAI includes a real-time pixel-art office simulation
where agents walk around, sit at desks, and type -- visually representing
what the AI system is doing. This is the hero differentiator: no other AI
framework lets you watch your agents work.

**Emotional hook:** "My AI employees have their own office. They walk to
their desks and start coding."

**Example topics:**
- First look at the pixel-art office -- agents arriving at work
- How the office reflects real pipeline state (agent working = character at
  desk typing)
- Customizing the office: furniture, layout, agent appearances
- The office as a monitoring dashboard -- why visualization beats logs
- Side-by-side: terminal output vs office view of the same directive

**Format affinity:** Video/screencast (office animation is inherently visual),
GIF carousel (animated frames showing agent movement)

### Pillar C: Developer Productivity (开发者效率提升)

**Core narrative:** gruAI is a real workflow improvement. It handles the full
pipeline -- triage, plan, audit, build, review -- so the developer focuses on
direction, not execution. Concrete before/after comparisons.

**Emotional hook:** "I gave it a one-line directive. It shipped a reviewed
feature in 20 minutes."

**Example topics:**
- Setup in 3 commands: `npm install gru-ai`, configure, run
- How the pipeline handles code review automatically
- Real example: from CEO directive to merged PR (timed)
- Why structured agent pipelines beat "just ask ChatGPT"
- gruAI vs manual development: time comparison on a real task

**Format affinity:** Carousel (before/after screenshots, setup guide slides),
screencast (live demo of directive execution)

### Key Leading Indicators

Track these weekly to detect problems early:

- **Save rate dropping below 3%:** Content is not reference-worthy. Add more
  cheat sheets, setup guides, comparison tables.
- **Comments below 3 per post:** CTAs are too generic. Use specific questions
  ("你的开发流程里最烦的一步是什么?").
- **Views stalling under 500:** Algorithm is not expanding distribution. Check
  posting consistency, hashtag relevance, visual quality.
- **GitHub stars not growing:** Content is entertaining but not converting.
  Add clearer CTAs with repo link, quickstart instructions.

## 4. Chinese Content Writing Guidance

### Tone & Voice

- **Conversational, not formal.** 小红书 is a peer platform. Write like a
  developer sharing with friends, not a company writing documentation.
- **First person.** Use 我 (I) and 我们 (we), not passive voice.
- **Show, don't tell.** "看，这是我的AI团队在开发" > "gruAI是一个先进的AI框架"
- **Self-deprecating humor is welcome.** "又被AI CTO打回了，她比我前同事还严格"
  -- this humanizes the agents.
- **Avoid:** Corporate language, buzzword soup, machine-translated stiffness.
  Read it aloud -- if it sounds like a press release, rewrite it.

### Key Technical Terms in Chinese

Use these consistently across all posts:

| English | Chinese | Notes |
|---------|---------|-------|
| AI agent | AI智能体 / AI Agent | 智能体 is more native, Agent is understood |
| Pipeline | 流水线 / Pipeline | 流水线 for Chinese audience, Pipeline in code |
| Directive | 指令 | Use consistently -- not 命令 or 任务 |
| C-suite | C-suite / 高管团队 | Keep English term, it's recognizable |
| CTO, CPO, CMO, COO | 直接用英文 | Do not translate titles |
| Code review | 代码审查 | Standard term |
| Framework | 框架 | Standard term |
| Open source | 开源 | Standard term |
| npm package | npm包 | Keep npm in English |
| Pixel art | 像素风 / 像素画 | 像素风 is more trendy |
| Autonomous | 自主 / 自动化 | 自主 for agency, 自动化 for automation |
| Screencast | 录屏演示 | Literal translation works |
| Triage | 分级 / 筛选 | Context-dependent |
| Audit | 审计 | Standard term |

### Writing Patterns That Work on 小红书

**Hook formulas (first line of post):**
- Challenge assumption: "ChatGPT只会回答问题，但如果AI能自己完成整个项目呢?"
- Visual surprise: "我的AI员工有自己的像素风办公室（附图）"
- Number hook: "3行命令，你的AI团队就开始上班了"
- Relatable pain: "每次代码审查都想摔键盘？让AI CTO帮你审"
- Behind-the-scenes: "做了一个月AI自动化公司，来汇报一下成果"

**CTA formulas (end of post):**
- Question: "你会用AI agent来做什么？评论区聊聊"
- Save prompt: "收藏这篇，下次配置的时候用得上"
- Share prompt: "觉得有用就分享给你的程序员朋友"
- Follow prompt: "关注我，每周分享AI自动化开发的新玩法"
- Action: "想试试的话，npm install gru-ai，3分钟就能跑起来"

**Formatting tips:**
- Use emoji sparingly -- 1--2 per paragraph max. 小红书 culture uses them
  but tech content should stay clean.
- Line breaks between every 2--3 sentences. Dense blocks get scrolled past.
- Bold key phrases by wrapping in brackets: 【重点】or 「关键」
- Number lists for sequential content (setup steps, comparison points)
- Keep paragraphs under 3 lines on mobile preview
